# 🍔 ReactFood App

> React 기반 음식 주문 웹 애플리케이션  
> **상태 관리 · 비동기 처리 · 클라이언트-서버 데이터 흐름**을 직접 설계하고 구현한 프로젝트입니다.

---

## 📌 프로젝트 소개

단순 UI 구현을 넘어,  
**전역 상태 관리 · 비동기 로딩 처리 · 주문 데이터 전송 흐름**을 고려하여  
실제 서비스와 유사한 구조를 설계하는 것을 목표로 했습니다.

백엔드 API와 통신하여 데이터를 불러오고,  
장바구니 상태를 관리하며,  
사용자 입력을 수집해 주문 데이터를 서버로 전송하는 전체 흐름을 구현했습니다.

---

## 🛠 Tech Stack

### 🎨 Frontend

- React
- Context API
- useState / useEffect
- Fetch API

### ⚙️ Backend

- Node.js
- Express
- REST API

---

## 🎯 기획 배경

기존의 간단한 React 프로젝트는 대부분  
정적인 UI 구현 또는 단순 상태 관리에 그치는 경우가 많았습니다.

이 프로젝트는 다음과 같은 고민에서 출발했습니다.

- 전역 상태는 어떻게 구조적으로 관리할 수 있을까?
- 비동기 데이터 요청 시 사용자 경험을 어떻게 개선할 수 있을까?
- 폼 데이터를 어떤 구조로 가공하여 서버에 전달하는 것이 좋을까?
- 실제 서비스처럼 장바구니 로직을 설계하려면 어떻게 해야 할까?

이러한 질문을 바탕으로  
**클라이언트-서버 데이터 흐름을 직접 설계하고 구현**했습니다.

---

## ✨ 주요 기능

- 음식 목록 조회 (GET)
- 장바구니 상품 추가 / 수량 증가 / 감소
- 총 금액 자동 계산
- 모달 기반 Cart & Checkout UI
- FormData 기반 주문 정보 수집
- 주문 객체 생성 후 POST 요청
- Loading / Error 상태 처리
- 불변성을 유지한 상태 업데이트
